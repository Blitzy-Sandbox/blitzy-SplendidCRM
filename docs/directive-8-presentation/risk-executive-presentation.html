Technical Debt Impact

COSO Principle 10 — Control Activities Assessment

1. Zero Testing Infrastructure Critical — COSO Principle 16, NIST SI
No unit tests, no integration tests, no E2E tests, no CI/CD, no static analysis. Cannot verify ANY internal control functions correctly; COSO Principle 16 FAIL.
Source: Repository-wide — zero test files found
2. MD5 Password Hashing Critical — COSO Principle 10, NIST IA-5
NIST-deprecated hash algorithm (MD5CryptoServiceProvider). Credential theft risk; SugarCRM backward compatibility trade-off.
Source: Security.cs:L393-406
3. 38 Unmanaged DLLs Critical — COSO Principle 9, NIST CM-3, CIS Control 2
Manual BackupBin2012/ management; no NuGet, no SBOM. 15 phantom HintPath references to non-existent BackupBin2022/BackupBin2025. Supply chain vulnerability; unable to verify library CVEs.
Source: SplendidCRM7_VS2017.csproj, BackupBin2012/
4. Legacy Client Coexistence Critical — COSO Principle 11, NIST SI
4 frontend clients: React 18.2.0, Angular ~13.3.0 (experimental), HTML5/jQuery 1.4.2–2.2.4, WebForms. SignalR version asymmetry: server v1.2.2 vs client v8.0.0 + v2.4.3. Expanded attack surface; jQuery 1.4.2 has known XSS vulnerabilities.
Source: React/package.json, Angular/package.json, BackupBin2012/
5. Monolithic Code Files Critical — COSO Principle 11, NIST CM
SplendidCache.cs: 11,582 LOC · Rest.svc.cs: 8,369 LOC · SplendidDynamic.cs: 7,458 LOC · Sql.cs: 4,082 LOC. High coupling; any change risks cascading failures across all systems.
Source: SplendidCRM/_code/SplendidCache.cs, Rest.svc.cs, SplendidDynamic.cs, Sql.cs
The combination of zero testing and monolithic architecture creates a scenario where defect correction cannot be safely validated — a Critical gap per COSO Principle 10 and NIST SI.
